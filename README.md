# SII-SLS 图像超分辨率工具

基于 **Semi-local Similarity** 的单图像 2x 超分辨率插值算法。

---

## 1. 如何运行本项目

### 环境准备

```bash
# 创建虚拟环境
python -m venv .venv
.venv\Scripts\activate   # Windows
# source .venv/bin/activate  # Linux/Mac

# 安装依赖
pip install -r requirements.txt
```

### 运行程序

```bash
python siisls_gui.py
```

### 没有 LR 和 GT 图像时的处理方法

本工具提供了 **"制作数据"** 功能，可以从一张高分辨率图像自动生成 LR 和 GT 图像对：

1. 点击界面上的 **"📊 制作数据"** 按钮
2. 选择一张高分辨率图像（任意尺寸）
3. 系统会自动：
   - 将原图裁剪为 8 的倍数尺寸（算法要求）
   - 保存为 GT 文件（`原图名_gt.png`）
   - 将图像按 2x 下采样生成 LR 文件（`原图名_lr.png`）

**生成的图像对尺寸关系：**
- GT（高分辨率）: `W × H`
- LR（低分辨率）: `W/2 × H/2`

### 使用生成的数据进行测试

1. 点击 **"实验图像 (LR)"** 按钮，选择 `_lr.png` 文件
2. 点击 **"对照图像 (GT)"** 按钮，选择 `_gt.png` 文件
3. 调整参数（默认参数通常效果较好）
4. 点击 **"🚀 开始超分辨率"** 开始处理
5. 处理完成后会自动计算 PSNR 和 SSIM 指标

## 2. 算法原理

本工具实现了基于 **半局部相似性 (Semi-local Similarity)** 的图像超分辨率算法，源自 ICASSP 2019 论文。

### 算法流程（4个阶段）

```
输入图像 (LR) 
    │
    ▼
┌─────────────┐
│  Stage 1    │ 初始插值 + 补丁重建 (10%)
│  (0-10%)    │ - 双三次插值生成初始HR
│             │ - 基于半局部相似性重建补丁
└─────────────┘
    │
    ▼
┌─────────────┐
│  Stage 2    │ 迭代优化 - 偶/奇像素分离 (20%)
│  (10-30%)   │ - 使用 K 近邻进行稀疏编码
│             │ - 迭代次数由 iter1 控制
└─────────────┘
    │
    ▼
┌─────────────┐
│  Stage 3    │ 全像素优化 (25%)
│  (30-55%)   │ - 使用所有像素而非仅 OO 相位
│             │ - 迭代次数由 iter2 控制
└─────────────┘
    │
    ▼
┌─────────────┐
│  Stage 4    │ DC 分量移除 + 最终重建 (45%)
│  (55-100%)  │ - 减去均值后进行补丁重建
│             │ - 迭代次数由 iter3 控制
└─────────────┘
    │
    ▼
 输出图像 (HR) - 尺寸为输入的 2 倍
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **补丁 (Patch)** | 图像的小块区域，用于局部相似性匹配 |
| **K 近邻 (K-NN)** | 在搜索区域内找 K 个最相似的补丁 |
| **稀疏编码** | 用近邻补丁的线性组合重建目标像素 |
| **相位模式** | 图像像素的奇偶位置关系（OO, OE, EO, EE） |
| **重叠相加** | 将重建的补丁合并回完整图像 |

### 为什么需要 GT（对照图像）？

- **PSNR**: 峰值信噪比，越高越好（通常 25-40 dB）
- **SSIM**: 结构相似性指数，越接近 1 越好（通常 0.7-1.0）
- 这两个指标用于量化评估超分辨率效果

## 3. 界面参数说明

### 图像选择区

| 控件 | 说明 |
|------|------|
| **对照图像 (GT)** | 高分辨率原始图像，用于计算精度指标。可不选。 |
| **实验图像 (LR)** | 低分辨率输入图像，**必须选择**。 |
| **输出目录** | 超分结果保存位置，默认在用户目录的 `SII_SLS_Output`。 |

### 参数设置区

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **n1 (补丁大小)** | 8 | 4-16 | Stage 1 使用的补丁边长（像素） |
| **n2 (补丁大小)** | 6 | 4-16 | Stage 2/3/4 使用的补丁边长 |
| **K (近邻数)** | 12 | 1-50 | 每个补丁搜索的相似补丁数量，越大越精细但越慢 |
| **sigma (平滑)** | 0.85 | 0.1-5.0 | 高斯核标准差，控制低通滤波强度 |
| **CW (权重)** | 500 | 100-2000 | 软阈值权重参数 |
| **迭代次数1** | 2 | 1-10 | Stage 2 迭代次数 |
| **迭代次数2** | 2 | 1-10 | Stage 3 迭代次数 |
| **迭代次数3** | 6 | 1-20 | Stage 4 迭代次数 |
| **实时预览** | ✓ | - | 是否在处理过程中显示预览 |
| **彩色处理** | ✓ | - | 是否对彩色图像进行 RGB 通道分别处理 |

### 进度显示区

| 控件 | 说明 |
|------|------|
| **总进度条 (绿色)** | 整体处理进度 0-100% |
| **阶段进度条 (蓝色)** | 当前阶段详细进度和状态文字 |
| **PSNR** | 峰值信噪比（绿色显示），需提供 GT 才有值 |
| **SSIM** | 结构相似性（橙色显示），需提供 GT 才有值 |

### 控制按钮

| 按钮 | 说明 |
|------|------|
| **📊 制作数据** | 从高分辨率图像生成 LR/GT 图像对 |
| **🚀 开始超分辨率** | 开始 2x 超分辨率处理 |
| **⏹️ 取消** | 中止正在进行的处理 |

---

## 4. 参数调优建议

### 追求速度
- 减小 `n1`、`n2`、`K` 值
- 减少 `iter1`、`iter2`、`iter3` 迭代次数

### 追求质量
- 增大 `K` 值（16-24）
- 增大 `iter3` 迭代次数（8-12）
- 适当增大 `sigma` 值

### 默认参数
适合大多数图像，质量与速度平衡较好。

---

## 5. 输出说明

处理完成后：
- 结果保存在设定的 **输出目录**
- 文件名为 `{原文件名}_sr.png`
- 如果提供了 GT，会显示 PSNR 和 SSIM 数值
